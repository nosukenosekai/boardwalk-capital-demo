# Working style

For any non-trivial or multi-step task in this repository, follow the
`hard-task-method` skill (`.claude/skills/hard-task-method/SKILL.md`):

1. Define what observable "done" looks like before starting.
2. Scout the actual code before planning; decompose into small,
   independently verifiable steps, riskiest first.
3. Verify every claim with outside evidence (run it, read it back,
   exercise it end-to-end) — never report predicted success as fact.
4. At each pause point, deliberately choose: continue, re-plan,
   escalate to the user, or stop — based on evidence, not momentum.
5. Report faithfully: what was done, what was verified, and what was
   deliberately left out.
