# Issue設計・実装担当割り当て（by Fable）

作成日: 2026-07-10 ／ 設計・レビュー統括: Fable (claude-fable-5)

## 運用ルール

1. 各Issueの設計はFableがIssue本文の「設計」セクションに記載する。実装者は設計に厳密に従う。
2. 実装担当は難易度・リスクに応じて Sonnet → Opus → Fable → 人間 の順で割り当てる。
   - **Sonnet**: 範囲が明確で定型的なタスク
   - **Opus**: 既存コードの挙動理解・回帰リスク管理が必要な中難度タスク
   - **Fable**: 回帰リスクが高い・設計判断を伴う高難度タスク
   - **人間**: 事業判断・法務判断・権利確認を伴うタスク
3. Opus / Sonnet のPRはFableがレビューする。受け入れ基準を満たさない場合は
   **上位モデルに再割り当てして再実行**する（Sonnet不合格→Opus、Opus不合格→Fable）。
4. PRのマージは行わない（リポジトリオーナーの判断）。
